# 通用资产租赁与催收管理系统：需求总纲

## 一、系统定位

一个以"合同"为核心的通用资产租赁与催收管理工具。

- **核心解决**：签合同、收租金、催欠款，这三个最累、最繁琐、最易出错的环节
- **第一个切入场景**是社区/村委的商铺租赁，但系统架构不绑定"房屋"，可扩展至车位、摊位、设备等任何可出租的资产
- **核心原则**：不改变人的工作习惯，只在背后自动完成记录、计算和提醒

---

## 二、五大核心理念

### 理念一：业务即录入，无痛冷启动

- 签合同时，资产信息和租户信息自动沉淀到系统里，无需提前建台账
- 新资产只填一个名称和月租金即可，面积等辅助信息以后慢慢补
- 用上一年，全部数据自然成型

### 理念二：合同是唯一锚点

- 所有租金计算、欠费判断、催缴触发，都以单份合同为独立单元
- 合同核心字段：开始日、结束日、月租金（或年租金）
- 系统自动算出合同应收总额，首月折半等线下商定可手动微调
- 合同模板由用户自行上传，资产字段跟随模板走，系统核心计算只依赖租期和租金

### 理念三：收款即记账，总额判欠费

- 不预设月付、季付、年付等缴费方式
- 每一笔收款关联到具体合同，系统累加"已收总额"
- 欠费判断只有一个标准：已收总额 < 合同应收总额

### 理念四：五级递进催缴，有节奏、分轻重

系统根据"钱用到哪天"，自动生成五级催缴清单：

| 等级 | 清单名称 | 触发条件 | 紧急程度 | 建议动作 |
|------|----------|----------|----------|----------|
| 一级 | 应缴预警 | 距"钱用完日"≤30天 | ⭐ | 列入观察，心中有数 |
| 二级 | 近期应缴提醒 | 距"钱用完日"≤7天 | ⭐⭐ | 电话或微信友好提醒 |
| 三级 | 逾期未缴催收 | 已超过"钱用完日" | ⭐⭐⭐ | 发正式催缴通知、上门 |
| 四级 | 到期预警 | 距"合同到期日"≤30天 | ⭐⭐⭐⭐ | 最后通牒，准备不续租 |
| 五级 | 已到期欠费追缴 | 已超合同到期日 | 🔴 | 进入追讨，法律途径 |

### 理念五：规范与灵活并存

- 合同本身规范，起止日期和金额清晰无歧义
- 合同期限灵活：一年期、短租几个月、跨年、对月、对年都支持
- 首月折半、1/3、2/3等线下商定，最终落实为合同上的起止日期和应收总额，系统不做限制
- 通过资产分组/标签，实现"按片区、按月份集中收租"的管理意图

## 三、三大日常操作入口

### 入口一：签新合同（数据积累的起点）

**工作人员的核心操作：**

1. **选资产**：输入资产名称，下拉选择已有的，或直接输入新名称（系统自动新建）
2. **录租户**：拍摄身份证正面，OCR自动识别填入姓名、身份证号、住址
3. **定合同**：
   - 填写合同开始日、结束日、月租金（或年租金，可互相换算）
   - 系统自动算出合同应收总额，工作人员确认（可微调）
4. **打印输出**：一键生成合同文件（套Word模板）+ 身份证复印件 + 三联收据

**系统后台自动完成：**

- 新资产自动创建（仅有名称和月租金）
- 新租户自动创建
- 合同档案建立，状态为"执行中"
- 资产和租户通过合同自动关联

### 入口二：收租金（日常最高频操作）

**工作人员的核心操作：**

1. **选合同**：输入租户姓名或资产名称，系统列出该租户/资产对应的正在执行的合同
2. **收款开票**：输入本次实收金额，系统自动显示"还差多少"
3. **打印收据**：一键生成三联收据，A4纸三等分，打印、撕开、交给租户

**系统后台自动完成：**

- 收款记录新增，关联到对应合同
- 合同"已收总额"更新
- 重新计算"钱用到哪天"
- 如果已收总额 ≥ 应收总额，合同自动标记为"已缴全"
- 如果仍是欠费状态，进入五级催缴体系的对应层级

### 入口三：看催缴清单（系统自动工作，无需人为操作）

系统每日自动刷新五张清单，按紧急程度排列：

| 清单 | 谁在上面 | 工作人员做什么 |
|------|----------|----------------|
| 应缴预警 | 钱快用完的人（30天内） | 心中有数，提前安排 |
| 近期应缴提醒 | 钱马上用完的人（7天内） | 友好提醒一下 |
| 逾期未缴催收 | 钱已经用完的人 | 正式催缴，发通知或上门 |
| 到期预警 | 合同快到期还没缴全的人 | 最后通牒，明确后果 |
| 已到期欠费追缴 | 合同已到期还欠钱的人 | 启动追讨程序，法律手段 |

每张清单附带建议动作，工作人员照单执行，不用自己判断催缴力度。

## 四、四大后台模块

| 模块 | 核心内容 |
|------|----------|
| 资产管理 | 资产列表（名称、月租金为核心字段，其他信息可选填、逐步完善）；历史租赁记录自动关联 |
| 租户管理 | 租户列表（身份证信息自动存入）；历史合同记录可追溯 |
| 合同管理 | 合同模板上传与字段映射；合同列表与状态流转（执行中/已缴全/欠费中/已到期）；每一次收款记录关联合同 |
| 催缴管理 | 五级清单自动生成与刷新；每份合同的催缴历史可追溯 |

---

## 五、系统架构总览

```
[签新合同] ──→ 自动沉淀 ──→ 资产档案 + 租户档案 + 合同档案
                    │                    │
[收租金]  ──→ 更新合同已收总额 ──→ 算出"钱用到哪天"
                    │                    │
                    │                    ↓
                    │     系统每日自动判断 ← 五级催缴规则
                    │                    │
                    ↓                    ↓
[看催缴清单] ←── 五张分级清单，附带建议动作
```

## 六、约束与边界

- 不改变人的工作习惯，只在背后自动完成记录、计算和提醒
- 合同模板由用户自行上传，系统不预设合同格式
- 不预设缴费方式（月付/季付/年付），仅以总额判断欠费
- 首月折半等特殊商定，通过调整合同应收总额处理，系统不做特殊逻辑
- 资产字段跟随合同模板走，系统核心计算仅依赖租期和租金
- 第一个切入场景：社区/村委商铺租赁，架构预留多资产类型扩展
